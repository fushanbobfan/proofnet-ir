# v0.8 intrinsic canonicalization design

## Scope

v0.8 computes a deterministic representative of the already documented
`Certificate.ProofNetEquivalent` relation without enumerating the factorial
link-order orbit. That relation preserves ordered conclusions, tensor/par
premise order, formula labels and polarity, and axiom endpoint orientation. It
quotients only bounded vertex renaming and literal link-list storage order.
This is deliberately not arbitrary graph isomorphism.

## Construction

`Certificate.intrinsicTraversalVertices` starts from the ordered conclusions.
For each stored formula occurrence it follows the unique producer selected
from the submitted links. Tensor and par premises are visited left before
right, following formula syntax; atoms end the walk. Duplicate removal is
fail-closed support for malformed inputs.

Each link then receives an orientation-sensitive owner:

- an axiom is owned by its stored left endpoint;
- a tensor or par is owned by its conclusion.

`intrinsicOrderedLinks` emits the unique owned link as owners occur in the
traversal. `intrinsicCanonicalize` applies the existing proved
first-occurrence vertex relabeling to the certificate with that link order.
`intrinsicCanonicalCode` uses the existing injective framed structural
encoder.

## Kernel theorem chain

On `Certificate.StructurallyWellFormed` inputs, Lean proves:

1. every in-bounds formula vertex occurs in the intrinsic traversal;
2. every traversal vertex is in bounds, so the traversal has exactly the
   formula-array size;
3. every submitted link has exactly one owner and submitted links are
   duplicate-free;
4. `intrinsicOrderedLinks` is exactly a permutation of submitted links;
5. the intrinsic canonical form is `ProofNetEquivalent` to its input;
6. equivalent inputs have equal intrinsic canonical forms;
7. canonical-form, structural-code, and typed-key equality are each equivalent
   to exactly `ProofNetEquivalent`.

Checker acceptance supplies structural well-formedness, yielding the public
checked-input iff theorems. The axiom audit pins the new logical boundary to
exactly `[propext, Classical.choice, Quot.sound]`, the same dependencies as the
existing MLL metatheory.

## Complexity and limits

No step enumerates link permutations. The current direct implementation scans
the link list while discovering owners/producers and uses list-based
first-occurrence indexing during relabeling. Its conservative bound is
`O(VL + V^2)` for `V` formula occurrences and `L` links, excluding the size of
serialized formula tokens. Because every occurrence stores its full formula,
the emitted character volume can be quadratic in formula depth.

The v0.8 benchmark generates and round-trips four structurally well-formed
identity certificates through 145 links in 120 ms aggregate on the recorded
Windows development run, below a five-second regression budget. A depth-64
diagnostic reached 1,292,168 token characters and was correctly rejected by
the independent one-million-character wire envelope; the qualified maximum is
therefore depth 48 / 145 links. This is a measured regression boundary, not an
asymptotic or production-latency claim.

The all-switchings proof-net checker remains exponential in the number of par
links, and executable sequentialization remains depth-sensitive. The
canonicalizer benchmark therefore uses the structural theorem domain rather
than disguising checker cost as canonicalizer cost.

## Wire and migration

The new envelope is:

- version: `proofnet-canonical-key-0.2`;
- canonicalization: `proofnet-equivalent-intrinsic-v1`;
- payload: the intrinsic framed token code;
- parser limits: 100,000 tokens and 1,000,000 aggregate token characters.

It is a new wire contract; v0.1 bytes are not reinterpreted. A
checker-accepted v0.3 certificate can be migrated semantically by recomputing
the intrinsic key. A bare v0.1 key cannot be converted without its source
certificate: the v0.1 factorial minimum and v0.2 intrinsic representative need
not use the same token sequence even though both are exact invariants of the
same relation.
