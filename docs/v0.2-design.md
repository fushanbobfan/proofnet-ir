# v0.2 derivation, serialization, dataset, and search design

## Arbitrary cut-free derivations

`CutFreeDerivation` is a first-order one-sided unit-free MLL syntax:

- `axiom` creates a dual atom pair;
- `tensor` combines two derivations and records one selected occurrence from
  each premise;
- `par` selects two occurrences from one premise;
- `exchange` records an explicit permutation of occurrence indices.

`build?` validates every selected position and exchange while constructing a
formula-occurrence fragment. `desequentialize?` turns the fragment into a
certificate. `desequentializeChecked?` additionally invokes the authoritative
reference checker and returns the certificate only with a Lean proof that
`check = true`. Invalid indices or permutations never produce a certificate.

The deterministic generator is not the definition of the supported language;
the public constructors accept arbitrary finite rule trees. The generator is a
reproducible corpus source that exercises varying resource positions, par
introduction, and explicit exchange.

## Canonical serialization boundary

The v0.2 serializer fixes the formula array as the vertex-identity contract and
normalizes:

1. axiom endpoint orientation by vertex number;
2. link order by a versioned numeric key;
3. conclusion order by vertex number;
4. JSON field representation and the version marker `0.2`.

It deliberately does not solve general labeled multigraph isomorphism. Two
certificates that differ only by arbitrary formula-array reindexing may retain
different v0.2 serializations. This boundary avoids presenting a deterministic
encoding as a stronger proof-identity theorem.

## Dataset trust path

The Lean emitter generates 250 valid certificates and three corruptions of
each. The Python regeneration script then checks schema validity, unique IDs,
label/provenance counts, and every label with the independent oracle before
writing or comparing the committed JSONL bytes and manifest hash.

## Focused baseline

`scripts/focused_search.py` implements the comparison baseline directly over
one-sided sequents. It eagerly applies invertible par rules, then explores each
tensor focus and every partition of the remaining linear resources. Sequents
are memoized modulo exchange by a canonical multiset ordering. Counters expose
states, cache hits, par steps, tensor choices, and resource partitions.

The baseline is exponential by design and is not trusted as a proof-net label
source. It exists to support matched experiments rather than to manufacture a
performance claim in advance.
